# Core API Service

This is the backend service for Project Nexus core platform (Epic 1). It uses FastAPI and SQLAlchemy ORM with SQLite persistence, exposing CRUD endpoints for buildings, floors, rooms, employees, assets, and work orders. The OpenAPI spec is automatically generated by FastAPI.

## Features

✅ **Epic 1 Implementation Complete**
- ADM-01: RBAC with 4 license levels (Self-Service, Work Process, Analysis, Process Owner)
- ADM-02: JWT authentication with secure password hashing
- ADM-03: System settings with terminology customization
- ADM-04: OpenAPI documentation (auto-generated in `/docs`)

## Architecture

- **Framework**: FastAPI 0.104.1
- **ORM**: SQLAlchemy 2.0.23
- **Database**: SQLite (file-based, persisted in `nexus.db`)
- **Auth**: JWT tokens with bcrypt password hashing
- **Models**: Pydantic for request/response, SQLAlchemy for persistence

## Running

```bash
cd services/core_api
python -m venv venv
source venv/bin/activate  # or .\venv\Scripts\activate on Windows
pip install -r requirements.txt
uvicorn app.main:app --reload
```

The API will be available at `http://localhost:8000` with auto-generated docs at `/docs`.

### Authentication
- Obtain token: POST `/login` with form data `username` and `password`
- Use returned bearer token for subsequent requests (`Authorization: Bearer <token>`)

## Database

- Uses SQLite for local development/testing
- Database file: `nexus.db` (auto-created on first run)
- Schema is created automatically on startup
- Can be easily migrated to PostgreSQL/MySQL by changing `DATABASE_URL`
